  <action>
    Create citation-tooltip.tsx with two components:

    1. `CitationRef` component:
       - Props: `{ id?: string; children: React.ReactNode; href?: string; "data-footnote-ref"?: boolean; footnoteText?: string }`
       - Renders as: `<TooltipProvider><Tooltip><TooltipTrigger asChild><sup className="cursor-help"><a ...>{children}</a></sup></TooltipTrigger><TooltipContent side="top" className="max-w-xs text-sm">{footnoteText || children}</TooltipContent></Tooltip></TooltipProvider>`
       - The superscript link uses classes: `text-primary/70 text-xs font-semibold no-underline hover:text-primary cursor-help`
       - If no footnoteText, still renders the superscript (just no tooltip or minimal tooltip)

    2. `CitationSection` component:
       - Props: `{ children: React.ReactNode; "data-footnotes"?: boolean; className?: string }`
       - Renders: `<section className="sr-only" aria-label="Footnotes">{children}</section>`
       - This hides the footnote definitions section visually (screen readers can still access it)

    3. `createFootnoteComponents` function:
       - Returns `Partial<Components>` with overrides for `sup` and `section`
       - `sup`: checks for `data-footnote-ref` attribute. If present, wraps children in CitationRef. If not, renders normal `<sup>`.
       - `section`: checks for `data-footnotes` attribute. If present, uses CitationSection. If not, renders normal `<section>`.
       - NOTE: react-markdown passes footnote definitions as children of the section. The footnote reference text needs to be extracted. Since remark-gfm renders footnote refs as `<sup><a href="#fn-1" data-footnote-ref>1</a></sup>` and definitions as `<section data-footnotes><ol><li id="fn-1"><p>text</p></li></ol></section>`, we need a strategy to connect them.
       - APPROACH: Use a React context or a simple approach -- the `section[data-footnotes]` contains the full text. For the `sup > a` elements, we cannot easily access the definition text from the ref. Instead, use CSS `:hover` + a data attribute approach OR use a simpler approach:
         - Parse the markdown before rendering to extract footnote definitions into a Map<string, string>
         - Pass this map via a React context to CitationRef components
         - `extractFootnotes(markdown: string): Map<string, string>` utility that regex-extracts `[^N]: text` patterns

    4. `extractFootnotes` utility function:
       - Input: raw markdown string
       - Output: `Map<string, string>` mapping footnote id to definition text
       - Regex: `/^\[\^([^\]]+)\]:\s*(.+)$/gm`
       - Export this for use in markdown.tsx

    5. `FootnoteContext`: React context providing `Map<string, string>` of footnote definitions.
       - `FootnoteProvider` wraps markdown content when footnotes are detected
       - CitationRef reads from this context using footnote href (e.g., `#user-content-fn-1` -> key `1`)

    Create tests in citation-tooltip.test.tsx using vitest + @testing-library/react. Wrap tests in FootnoteProvider with test data.
  </action>
