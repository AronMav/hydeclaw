Fix silent error swallowing in channels bridge HTTP methods and strip @-mention text from LLM context in Telegram group mode.

Output: Bridge methods that throw on failure with descriptive messages; Telegram driver strips @botname from text before sending to core.
</objective>

    ```typescript
    let text = msg.text ?? msg.caption ?? "";
    // Strip @botname from group messages so LLM sees clean text
    if (isGroup && botUsername) {
      text = text.replace(new RegExp(`@${botUsername}\\b`, 'gi'), '').trim();
    }
    ```

    This must happen AFTER the mention detection check (which uses the raw text) but BEFORE the text is passed to dispatchOrQueue / debounce / processMessage.

    The regex uses `\b` word boundary to avoid partial matches and `gi` for case-insensitive global replace. The `.trim()` handles leading/trailing whitespace left after removal.
