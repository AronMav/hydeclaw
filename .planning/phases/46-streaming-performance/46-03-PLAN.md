  <what-built>
    Three streaming performance improvements:
    1. PERF-01: rAF throttle (already live) — coalesces rapid SSE deltas to ~20fps, verified by automated test
    2. PERF-02: Stable block keys — unchanged markdown blocks no longer unmount/remount when the last block grows
    3. PERF-03: Deferred syntax highlighting — code blocks show plain monospace during streaming; Shiki fires only after closing fence (```) arrives or stream ends
  </what-built>
  <how-to-verify>
    **Prerequisites:** Start the dev server: `cd ui && npm run dev` (port 3000)

    **Test 1 — PERF-01: Frame rate during long stream**
    1. Open browser DevTools → Performance tab
    2. Open chat: `http://localhost:3000/chat`
    3. Send a message that triggers a long response (e.g., "Write a 500-word essay on any topic")
    4. During streaming: observe that the page does not jank — scrolling should feel smooth
    5. Expected: frame rate stays above 50fps in Performance tab during active streaming
    6. Also acceptable: visual smoothness without DevTools measurement

    **Test 2 — PERF-02: No block flickering during streaming**
    1. Send a message that triggers a multi-paragraph response (e.g., "List 5 programming best practices, one paragraph each")
    2. Watch the already-rendered paragraphs as new text arrives
    3. Expected: earlier paragraphs do NOT flash, repaint, or show any visual change while the last paragraph grows
    4. Failure sign: any "blink" or flicker on settled text blocks

    **Test 3 — PERF-03: Deferred syntax highlighting**
    1. Send a message that includes a code block (e.g., "Show me a JavaScript function with 10 lines")
    2. While streaming the code block:
       - Expected: code block shows plain monospace text (no colors)
       - Expected: NO copy button visible while code is streaming
    3. When the closing fence (```) arrives:
       - Expected: syntax highlighting appears (colors applied)
       - Expected: copy button appears
    4. After streaming completes: code should be fully highlighted with copy button visible
    5. Failure sign: colors flashing/changing mid-stream, or highlighting never appearing after stream

    **Test 4 — Regression: toggle streaming**
    1. Start a new chat and send a message
    2. While streaming, click Stop
    3. Send another message and let it complete
    4. Expected: highlighting appears correctly on the second message's code blocks
    5. Expected: no console errors during the above sequence
  </how-to-verify>
