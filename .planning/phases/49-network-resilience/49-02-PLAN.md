**A. sse-events.ts — parseSSELines already returns all lines including "id: 42".**
The function returns all lines split by newline. The callers (processSSEStream, SseConnection) currently skip non-"data:" lines with `if (!line.startsWith("data:")) continue`. The id: lines ARE returned by parseSSELines — they're just ignored.

No change needed to `parseSSELines` itself. But add a small helper:
```typescript
/** Extract the event ID from an SSE "id:" line. Returns null if not an id line. */
export function extractSseEventId(line: string): string | null {
  if (line.startsWith("id:")) {
    return line.slice(3).trim() || null;
  }
  return null;
}
```

Add tests for `extractSseEventId` in `sse-parsing.test.ts`:
- `extractSseEventId("id: 42")` returns `"42"`
- `extractSseEventId("id:42")` returns `"42"`
- `extractSseEventId("id:")` returns `null` (empty)
- `extractSseEventId("data: foo")` returns `null`
