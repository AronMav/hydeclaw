# Canvas — визуальное отображение контента

## Когда использовать Canvas

Используй инструмент `canvas` когда пользователь просит:
- Визуализировать данные (графики, диаграммы, charts)
- Показать dashboard или аналитику
- Нарисовать таблицу с форматированием
- Отобразить что-то визуально (не текстом)
- Создать интерактивный контент
- Показать HTML-страницу или виджет
- Вывести что-то "в canvas" или "на canvas"

## Как вызвать Canvas

```json
{
  "action": "present",
  "content_type": "html",
  "title": "Заголовок панели",
  "content": "<полный HTML с inline CSS и JS>"
}
```

## Правила

1. **action="present"** — показать контент в canvas панели
2. **content_type="html"** — всегда HTML для визуального контента
3. **content** — полная самодостаточная HTML-страница с inline стилями и скриптами
4. **title** — заголовок для canvas панели
5. **Максимальный размер** — 5MB
6. **ОБЯЗАТЕЛЬНО** — после canvas всё равно напиши текстовое резюме в чат. Пустой ответ = провал.

## Шаблон HTML для canvas

Всегда используй тёмную тему (background: #1a1a2e, color: #e0e0e0) чтобы соответствовать UI HydeClaw.

Для графиков подключай Chart.js через CDN:
`<script src="https://cdn.jsdelivr.net/npm/chart.js"></script>`

Для сложных визуализаций можно использовать D3.js, Mermaid, или любую CDN-библиотеку.

## Другие действия canvas

- **action="clear"** — очистить canvas
- **action="run_js"** — выполнить JS в текущем canvas (content = JS код)
- **action="snapshot"** — получить скриншот текущего canvas

## Когда НЕ использовать Canvas

- Для простого текста — пиши прямо в чат
- Для коротких списков — markdown в чате достаточно
- Для изображений из инструментов — они показываются inline автоматически
